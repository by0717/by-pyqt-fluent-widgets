# 输入框扫描事件处理 - 功能更新

## 🎯 更新内容

**日期**: 2026年1月27日  
**版本**: 1.1  
**更新**: 为每个输入框添加扫描枪事件处理

---

## ✨ 核心改进

### 功能说明

为每个窗口的 `mac_edit` 输入框都安装了扫描枪事件处理器（eventFilter）。

**现在支持的操作**：
- ✅ 焦点在输入框中时，按 `#1-#8` 直接跳转到对应窗口
- ✅ 跳转时自动清空当前输入框
- ✅ 跳转后自动设置焦点到目标窗口的输入框
- ✅ 无需手动切换焦点或点击其他地方

---

## 📝 改动详情

### 1. ScannerManager.bind_input()

**之前**:
```python
# 不为输入框安装eventFilter
```

**现在**:
```python
# ✅ 为每个输入框都安装eventFilter
widget.installEventFilter(self)
```

**优势**:
- 每个输入框都能独立接收扫描事件
- 无论焦点在哪，都能正确识别跳转指令

### 2. ScannerManager.eventFilter()

**改进**:
```python
def eventFilter(self, obj, event):
    """事件过滤器 - 处理来自parent和输入框的键盘事件"""
    # 现在能处理来自parent和所有input_widgets的事件
    # 当识别到跳转指令时，拦截特殊字符防止显示在输入框
```

**作用**:
- 拦截 `#` 字符，防止显示在输入框中
- 拦截数字键（跳转指令的一部分），防止显示
- 拦截回车键，正确处理输入完成

### 3. 优先级规则

```
焦点在输入框时，优先级顺序：

1️⃣ 跳转指令 (#1-#8)     → 最高优先级
   - 立即清空输入框并跳转
   - 字符不会显示在输入框
   
2️⃣ MAC地址输入           → 中等优先级
   - 正常填充到输入框
   - 按回车或超时后处理
   
3️⃣ 其他字符             → 低优先级
   - 正常输入处理
```

## 🔄 工作流程

### 场景：在输入框中按 #2

```
步骤 1: 焦点在窗口1的mac_edit，用户按#2
        ↓
步骤 2: mac_edit的eventFilter接收KeyPress事件
        ↓
步骤 3: ScannerManager.eventFilter()处理
        - 调用ScannerHandler.handle_key_event()
        - 识别"#2"为跳转指令
        ↓
步骤 4: ScannerManager._handle_jump(1)处理
        - 清空窗口2的mac_edit: widget.clear()
        - 设置焦点: widget.setFocus()
        - 标记等待状态
        ↓
步骤 5: ExecutionInterface.on_scanner_jump()处理
        - 打印日志
        ↓
结果: 焦点转移到窗口2，输入框已清空，可扫描新MAC
```
